DNS (Domain Name System)

DNS is a hierarchical and distributed naming system that translates domain names into IP addresses, allowing users to access websites and online resources using easy-to-remember domain names instead of difficult-to-remember IP addresses.

How DNS Works

  1. User enters a domain name into their browser.
  2. Browser checks local cache for IP address.
  3. If not found, request is sent to DNS resolver.
  4. DNS resolver queries root DNS server, which responds with TLD server IP.
  5. TLD server responds with authoritative DNS server IP.
  6. Authoritative DNS server responds with IP address of web server.
  7. Browser connects to web server and loads webpage.

Types of DNS

  • Recursive DNS: A recursive DNS server is responsible for resolving a domain name into an IP address.
  • Authoritative DNS: An authoritative DNS server is responsible for providing the IP address of a domain name.
  • Public DNS: A public DNS server is a DNS server that is open to the public and can be used by anyone.
  • Private DNS: A private DNS server is a DNS server that is only accessible within a private network.

Types of DNS Queries

  • Recursive Query: In this query, the DNS client wants the DNS server to respond with the requested resource record or an error message.
  • Iterative Query: In this query, the DNS client wants the best possible answer from the DNS server.
  • Non-Recursive Query: In this query, the DNS resolver queries a DNS server for a record that exists in its cache.

DNS Caching and TTL (Time-to-Live)

DNS caching stores DNS records locally to reduce network traffic. TTL is the time a DNS record is cached before expiring.

Types of DNS Records

  • A Record (Address Record): Maps a domain name to an IP address.
  • AAAA Record (IPv6 Address Record): Maps a domain name to an IPv6 address.
  • MX Records (Mail Exchange Records): Specifies the mail server for a domain.
  • CNAME Records (Canonical Name Records): Maps an alias to a canonical domain name.
  • TXT Record (Text Record): Stores text-based data for a domain.
  • NS Record (Name Server Record): Specifies the name servers for a domain.
  • SRV Record (Service Record): Specifies the location of a service.
  • PTR Record (Pointer Record): Maps an IP address to a domain name.

Advantages of DNS

  • Easy to remember domain names
  • Flexibility in changing IP addresses
  • Scalability in handling large traffic

Disadvantages of DNS

  • Security risks (DNS spoofing, DNS tunneling)
  • Performance issues (latency, caching)
  • Dependence on DNS servers
