Re-Rendering Components in React JS

Overview

Re-rendering is the process of updating a React component's UI when its state or props change. React automatically handles re-rendering for you, but understanding how it works can help you optimize your app's performance.

When Does Re-Rendering Happen?

  • State Change: When a component's state changes using setState or useState.
  • Props Change: When a component's props change.
  • Parent Re-Render: When a parent component re-renders, its child components will also re-render.

How to Prevent Unnecessary Re-Renders

  • Use React.memo: Wrap your component with React.memo to memoize it and prevent re-renders when props don't change.
  • Use useCallback: Memoize functions with useCallback to prevent them from changing on every render.
  • Use useMemo: Memoize values with useMemo to prevent expensive calculations on every render.
Tip: Use React DevTools to profile your app and identify performance bottlenecks.

Example Use Cases

Example 1: Preventing Re-Renders with React.memo


import React from 'react';

const MyComponent = React.memo((props) => {
  console.log('Rendered!');
  return <div>Hello, {props.name}!</div>;
});

Example 2: Preventing Re-Renders with useCallback


import React, { useState, useCallback } from 'react';

function MyComponent() {
  const [count, setCount] = useState(0);
  const handleClick = useCallback(() => {
    console.log('Button clicked!');
  }, []);

  return <button onClick={handleClick}>Click me ({count})</button>;
}

Example 3: Preventing Re-Renders with useMemo


import React, { useState, useMemo } from 'react';

function MyComponent() {
  const [count, setCount] = useState(0);
  const expensiveCalculation = useMemo(() => {
    // expensive calculation here
    return count * 2;
  }, [count]);

  return <div>Expensive calculation: {expensiveCalculation}</div>;
}

Best Practices

  • Avoid unnecessary re-renders: Use React.memo, useCallback, and useMemo to prevent unnecessary re-renders.
  • Use immutable state: Avoid mutating state directly, and instead use setState or useState to update state.
  • Profile your app: Use React DevTools to profile your app and identify performance bottlenecks.
