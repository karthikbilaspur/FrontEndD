React Virtual DOM

What is Virtual DOM?

The Virtual DOM is a lightweight in-memory representation of the real DOM. It's a programming concept used by React to optimize UI updates.

How Virtual DOM Works

  • When state changes, React updates the Virtual DOM.
  • React compares the updated Virtual DOM with the previous one (diffing).
  • Calculates minimal changes needed to update the real DOM.
  • Applies only those changes to the real DOM (reconciliation).
State Change
Update Virtual DOM
Diffing
Update Real DOM (minimal changes)

Benefits

  • Performance: Minimizes costly DOM operations.
  • Efficiency: Batch updates for better rendering.
  • Declarative UI: Focus on what to render, not how.

Virtual DOM vs Real DOM

  • Virtual DOM: Lightweight, in-memory, fast diffing.
  • Real DOM: Browser API, slower updates, heavy operations.
// Virtual DOM (simplified)
const vDOM = {
type: 'div',
props: { id: 'app' },
children: [...]
};
// Real DOM
const element = document.createElement('div');
element.id = 'app';

Reconciliation (Diffing Algorithm)

  • Compares elements by type and key.
  • Recursively updates children.
  • Uses keys to optimize list updates.
// Example with keys
<ul>
{items.map(item => <li key={item.id}>{item.name}</li>)}
</ul>

When to Use Keys

  • Lists or dynamic elements.
  • Ensure unique key prop for siblings.
<TodoItem key={todo.id} todo={todo} />

Key Points

  • Virtual DOM optimizes updates.
  • React handles diffing and reconciliation.
  • Use keys for efficient list rendering.
