React events are handled similarly to regular HTML events, but with some key differences. They're synthetic, meaning they're a wrapper around native browser events, ensuring cross-browser compatibility and optimized performance.

Key Features of React Events

  • Synthetic Events: A unified layer over native browser events, providing consistent behavior across browsers.
  • CamelCase Naming: Event names are camelCased (e.g., onClick instead of onclick).
  • Event Handlers as Props: Event handlers are passed as props to components.

Common React Events

  • Mouse Events: onClick, onMouseDown, onMouseUp, onMouseEnter, onMouseLeave
  • Keyboard Events: onKeyDown, onKeyUp, onKeyPress
  • Focus Events: onFocus, onBlur
  • Form Events: onChange, onSubmit, onReset

Example


import React from 'react';

function handleClick() {
  console.log('Button clicked!');
}

function MyButton() {
  return <button onClick={handleClick}>Click me!</button>;
}

Event Object

The event object passed to event handlers is a synthetic event, with properties like target, currentTarget, preventDefault(), and stopPropagation().

Best Practices

  • Use event.persist() when accessing event properties asynchronously.
  • Memoize event handlers with useCallback to prevent unnecessary re-renders.
