What is React Hook?

React Hook is a feature in React that allows you to use state and other React features in functional components. Hooks were introduced in React 16.8 as a way to use state and other React features in functional components, which were previously only available in class components.

Code

Types of React Hooks

  • State Hooks: useState, useReducer
  • Effect Hooks: useEffect, useLayoutEffect
  • Context Hooks: useContext
  • Ref Hooks: useRef
  • Performance Hooks: useMemo, useCallback
  • Custom Hooks: useYourHookName

Major Functions of React Hooks

  • useState: Returns a state value and an update function to update the state.
  • useEffect: Runs a function after rendering a component, and can be used to handle side effects.
  • useContext: Returns the value of a context.
  • useRef: Returns a reference to a DOM node or a value that persists across renders.
  • useMemo: Memoizes a value so it's not recomputed on every render.
  • useCallback: Memoizes a function so it's not recreated on every render.

Examples

useState

import React, { useState } from 'react';
function Counter() {
const [count, setCount] = useState(0);
return (
<div>
<p>Count: {count}</p>
<button onClick={() => setCount(count + 1)}>Increment</button>
</div>
);
}

function Timer() {
const [seconds, setSeconds] = useState(0);
useEffect(() => {

}, 1000);

useRef

import React, { useRef } from 'react';
function FocusInput() {
const inputRef = useRef(null);
const handleFocus = () => {
inputRef.current.focus();
};
return (
<div>
<input ref={inputRef} type="text" />
<button onClick={handleFocus}>Focus Input</button>
</div>
);
}

useCallback

import React, { useCallback } from 'react';
function Button() {
const handleClick = useCallback(() => {

Class Components vs Hooks

Feature Class Components Hooks
Syntax Uses JavaScript classes and extends React.Component Uses functional components and React hooks
State Management Uses this.state and this.setState Uses useState hook
Lifecycle Methods Uses lifecycle methods like componentDidMount, componentDidUpdate, etc. Uses useEffect hook
Context API Uses this.context and static contextType Uses useContext hook
Reusability Less reusable due to tight coupling with class syntax More reusable due to functional syntax and hooks
Complexity More complex due to class syntax and lifecycle methods Less complex due to functional syntax and hooks
Optimization Requires manual optimization using shouldComponentUpdate Automatic optimization using React.memo and useCallback
Testing More difficult to test due to class syntax and lifecycle methods Easier to test due to functional syntax and hooks
