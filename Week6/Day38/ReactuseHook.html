useEffect Hook

The useEffect hook is a way to handle side effects in functional components. Side effects are actions that can affect other components and cannot be done during rendering, such as setting timers, making API calls, or manually changing the DOM.

Code

How it works

The useEffect hook takes two arguments: a function to run and an array of dependencies. The function is run after the component is rendered, and the dependencies array determines when the function should be re-run.

Syntax

useEffect(() => {
// code to run
}, [dependencies]);
Code

Example

import { useEffect, useState } from "react";
function TimerComponent() {
const [count, setCount] = useState(0);
useEffect(() => {
const interval = setInterval(() => {
setCount((prev) => prev + 1);
}, 1000);
Code
//Cleanup function to clear the interval
return () => {
  clearInterval(interval);
  console.log("Interval cleared");
};
}, []); // Runs only once when the component mounts
return <h1>Timer: {count} seconds</h1>;
}
export default TimerComponent;
Code

Explanation

In this example, the useEffect hook is used to set up a timer that increments the count state every second. The cleanup function is used to clear the interval when the component is unmounted.

Dependencies Array

The dependencies array is used to determine when the effect should be re-run. If the array is empty, the effect is only run once when the component mounts. If the array contains values, the effect is re-run whenever any of the values change.

Common Use Cases

  • Setting timers or intervals
  • Making API calls
  • Manually changing the DOM
  • Setting up event listeners
