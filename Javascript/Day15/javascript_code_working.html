JavaScript Code Execution

JavaScript code execution is a two-phase process: parsing and execution.

Code

Parsing

When a JavaScript file is loaded, the browser's JavaScript engine parses the code, checking for syntax errors and creating an Abstract Syntax Tree (AST).

Execution

The JavaScript engine executes the code in the following order:

  • Global Execution Context: The global execution context is created, and the code is executed line by line.
  • Function Execution Context: When a function is called, a new execution context is created, and the function's code is executed.
  • Variable Hoisting: Variables declared with var are "hoisted" to the top of their scope, while let and const are not.

Browser Developer Mode

Most modern browsers come with a built-in developer mode, which provides tools for debugging and testing JavaScript code.

  • Console: A JavaScript console where you can execute code and see output.
  • Sources: A list of all loaded JavaScript files, with options to set breakpoints and inspect code.
  • Debugger: A debugger that allows you to step through code, inspect variables, and set breakpoints.

Execution Context

An execution context is an object that contains information about the current execution environment.

  • Variable Object: An object that stores variables, functions, and arguments.
  • Scope Chain: A chain of scope objects that determines variable access.
  • This: A reference to the current execution context's this value.

Example

	
		console.log(a); // undefined
		var a = 10;
		function foo() {
			console.log(b); // ReferenceError: b is not defined
			let b = 20;
		}
		foo();
