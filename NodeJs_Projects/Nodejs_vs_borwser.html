Node.js vs Browser

Node.js

  • Runs on the server-side
  • Built on Chrome's V8 JavaScript engine
  • Has access to the file system, network, and other system resources
  • Uses the CommonJS module system (require, module.exports)
  • Supports asynchronous I/O operations (e.g., fs, net)
  • No DOM (Document Object Model) or browser-specific APIs (e.g., window, document)

Browser

  • Runs on the client-side (user's web browser)
  • Built on various JavaScript engines (e.g., V8, SpiderMonkey, JavaScriptCore)
  • Has access to the DOM and browser-specific APIs (e.g., window, document)
  • Uses the ES6+ module system (import, export)
  • Supports asynchronous I/O operations (e.g., XMLHttpRequest, fetch)
  • Limited access to system resources (e.g., no file system access)

Key differences

  • Environment: Node.js runs on the server, while browsers run on the client.
  • Module system: Node.js uses CommonJS, while browsers use ES6+ modules.
  • System access: Node.js has access to system resources, while browsers have limited access.
  • APIs: Node.js has APIs for server-side tasks (e.g., fs, net), while browsers have APIs for client-side tasks (e.g., DOM, XMLHttpRequest).

Example: Making an HTTP request

Node.js (using http module)

const http = require('http');
http.get('https://example.com', (res) => {
  console.log(res.statusCode);
});

Browser (using fetch API)

fetch('https://example.com')
  .then((res) => console.log(res.status))
  .catch((err) => console.error(err));
